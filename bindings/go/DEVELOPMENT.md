# Developing The SlateDB Go Binding

`bindings/go/uniffi/slatedb.go` and `bindings/go/uniffi/slatedb.h` are
generated by `uniffi-bindgen-go`. `bindings/go/uniffi/cgo_flags.go`,
`bindings/go/go.mod`, `bindings/go/README.md`, and this file are maintained by
hand.

## Prerequisites

- Rust toolchain from this repository
- Go 1.25.0 or newer
- CGO enabled
- a working C toolchain
- `uniffi-bindgen-go` `0.7.0+v0.31.0`

Install the generator with:

```bash
cargo install uniffi-bindgen-go --git https://github.com/NordSecurity/uniffi-bindgen-go --tag v0.7.0+v0.31.0
```

## Regenerate Bindings

From the repository root:

```bash
./scripts/generate-go-uniffi.sh
```

The script builds `slatedb-uniffi`, runs `uniffi-bindgen-go`, copies the
generated Go package into `bindings/go/uniffi`, and applies the async error
workaround needed by `uniffi-bindgen-go v0.7.0+v0.31.0`.

## Validate Changes

From the repository root:

```bash
./scripts/generate-go-uniffi.sh
git diff --exit-code -- bindings/go
```

Then run Go tests from `bindings/go`:

```bash
cd bindings/go
CGO_ENABLED=1 CGO_LDFLAGS="-L$(pwd)/../../target/debug" \
LD_LIBRARY_PATH="$(pwd)/../../target/debug:${LD_LIBRARY_PATH}" \
DYLD_LIBRARY_PATH="$(pwd)/../../target/debug:${DYLD_LIBRARY_PATH}" \
go test ./...
```

## Local Runtime Linking

The generated Go package links against the shared library produced by
`slatedb-uniffi`. For local development, make sure your runtime loader can find
the library from `target/debug` or `target/release`.

Examples:

```bash
export LD_LIBRARY_PATH="$(pwd)/target/debug:$(pwd)/target/release:${LD_LIBRARY_PATH}"
export DYLD_LIBRARY_PATH="$(pwd)/target/debug:$(pwd)/target/release:${DYLD_LIBRARY_PATH}"
```
